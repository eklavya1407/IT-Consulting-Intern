# Client Presentation Guidelines
*Notes from IT Consultant Internship | Alankaeon Consulting | December 2025*

---

## Core Principles Followed

### 1. Know Your Audience
- Technical audience → include specs, architecture, implementation details
- Business/non-technical audience → focus on outcomes, cost, timeline, risk
- Mixed audience → lead with business impact, back it with technical evidence

### 2. Slide Structure Used

| Slide | Purpose |
|---|---|
| Cover Slide | Company name, project title, date |
| Agenda | 3-5 bullet overview of what will be covered |
| Problem Statement | What challenge are we solving? |
| Research Findings | Data, comparisons, market context |
| Recommended Solution | What we recommend and why |
| Implementation Plan | Steps, timeline, responsibilities |
| Q&A / Next Steps | Open discussion and action items |

### 3. Design Rules Applied
- Max 5-6 bullet points per slide
- One key message per slide
- Use charts instead of tables where possible
- Consistent font, color, and spacing throughout
- Company logo on cover and last slide

### 4. Data Visualization Tips
- Bar charts for comparisons
- Line charts for trends over time
- Pie charts only for simple proportions (max 4-5 segments)
- Always label axes and include data source

---

## Common Mistakes to Avoid
- Too much text on one slide (death by bullet points)
- Inconsistent formatting across slides
- Using jargon the client won't understand
- No clear "so what?" or recommendation at the end
